## <span class="api-section-title" id="stripe-webhook-endpoint">Stripe Webhook Endpoint</span>

### Webhook Handling for Stripe Payments and Connect Processes

The webhook endpoint located in the **stripe/webhook/route.ts** file handles events triggered by Stripe, such as payment and subscription updates.

```typescript
<code class="api-code">
import { NextRequest, NextResponse } from 'next/server'
import { headers } from 'next/headers'
import Stripe from 'stripe'
import { stripe } from '@/lib/stripe'
import { subscriptionCreated } from '@/lib/stripe/stripe-actions'

const stripeWebhookEvents = new Set([
  'product.created',
  'product.updated',
  'price.created',
  'price.updated',
  'checkout.session.completed',
  'customer.subscription.created',
  'customer.subscription.updated',
  'customer.subscription.deleted',
])

export async function POST(req: NextRequest) {
  let stripeEvent: Stripe.Event
  const body = await req.text()
  const sig = headers().get('Stripe-Signature')
  const webhookSecret =
    process.env.STRIPE_WEBHOOK_SECRET_LIVE ?? process.env.STRIPE_WEBHOOK_SECRET
  try {
    if (!sig || !webhookSecret) {
      console.log(
        'üî¥ Error Stripe webhook secret or the signature does not exist.'
      )
      return
    }
    stripeEvent = stripe.webhooks.constructEvent(body, sig, webhookSecret)
  } catch (error: any) {
    console.log(`üî¥ Error ${error.message}`)
    return new NextResponse(`Webhook Error: ${error.message}`, { status: 400 })
  }

  try {
    if (stripeWebhookEvents.has(stripeEvent.type)) {
      const subscription = stripeEvent.data.object as Stripe.Subscription
      if (
        !subscription.metadata.connectAccountPayments &&
        !subscription.metadata.connectAccountSubscriptions
      ) {
        switch (stripeEvent.type) {
          case 'customer.subscription.created':
          case 'customer.subscription.updated': {
            if (subscription.status === 'active') {
              await subscriptionCreated(
                subscription,
                subscription.customer as string
              )
              console.log('CREATED FROM WEBHOOK üí≥', subscription)
            } else {
              console.log(
                'SKIPPED AT CREATED FROM WEBHOOK üí≥ because subscription status is not active',
                subscription
              )
              break
            }
          }
          default:
            console.log('üëâüèª Unhandled relevant event!', stripeEvent.type)
        }
      } else {
        console.log(
          'SKIPPED FROM WEBHOOK üí≥ because subscription was from a connected account not for the application',
          subscription
        )
      }
    }
  } catch (error) {
    console.log(error)
    return new NextResponse('üî¥ Webhook Error', { status: 400 })
  }
  return NextResponse.json(
    {
      webhookActionReceived: true,
    },
    {
      status: 200,
    }
  )
}
</code>
```

## <span class="api-section-title" id="stripe-api-endpoints">Stripe API Endpoints</span>

### Stripe API Endpoints Overview

The Stripe API endpoints are organized under the **stripe/** directory, each handling specific functionalities such as creating checkout sessions, customers, and subscriptions.

- **create-checkout-session/route.ts**: Endpoint for creating Stripe checkout sessions.
- **create-customer/route.ts**: Endpoint for creating Stripe customers.
- **create-subscription/route.ts**: Endpoint for creating Stripe subscriptions.

These endpoints utilize the Stripe library and handle requests related to payment processing and customer management.

## <span class="api-section-title" id="uploadthing-api">UploadThing API</span>

### UploadThing API Overview

The UploadThing API, located in the **uploadthing/** directory, provides endpoints for handling file uploads and core functionalities of the "UploadThing" service.

- **core.ts**: Core functionality and utilities for the UploadThing service.
- **route.ts**: API endpoints for managing file uploads and related operations.

This API module integrates seamlessly with the main application to facilitate file management and data storage capabilities.
