<div class="core-container">
  <h2 class="core-section-title" id="root-directory">Root Directory</h2>
  <p class="core-description">
    The root directory of the NLP SAAS web app contains the essential files and folders that are required for the app to function correctly.
  </p>

  <pre class="core-code">
  ├── public
  ├── src
  ├── package.json
  ├── astro.config.mjs
  ├── tsconfig.json
  └── README.md
  </pre>

  <p class="core-description">
    The root directory includes configuration files such as `package.json`, `astro.config.mjs`, and `tsconfig.json`, along with the `public` and `src` directories which hold the public assets and source code respectively.
  </p>

  <h2 class="core-section-title" id="source-directory-src">Source Directory (`src`)</h2>
  <p class="core-description">
    The `src` directory is the main directory for the app's source code. It includes all the components, pages, and utility functions used in the app.
  </p>

  <pre class="core-code">
  ├── src
  │   ├── components
  │   ├── pages
  │   ├── layouts
  │   └── utils
  </pre>

  <p class="core-description">
    The `components` directory contains reusable UI components, the `pages` directory holds all the page components, the `layouts` directory includes layout components, and the `utils` directory stores utility functions.
  </p>

  <h3 class="core-subtitle">Components</h3>
  <p class="core-description">
    The `components` directory contains reusable components that can be used throughout the app. These components help in building the UI efficiently and consistently.
  </p>

  <h3 class="core-subtitle">Pages</h3>
  <p class="core-description">
    The `pages` directory includes all the page components that represent different routes in the app. Each file in this directory corresponds to a specific route.
  </p>

  <h3 class="core-subtitle">Layouts</h3>
  <p class="core-description">
    The `layouts` directory holds layout components that define the overall structure of the app's pages. These components help in maintaining a consistent layout across different pages.
  </p>

  <h3 class="core-subtitle">Utils</h3>
  <p class="core-description">
    The `utils` directory contains utility functions that can be used throughout the app. These functions help in performing common tasks and keeping the codebase organized.
  </p>
</div>

  <h2 class="core-section-title" id="database-directory">Database Directory</h2>
  <p class="core-description">
    The `database` directory contains scripts and configurations related to the app's database. It ensures that the database setup and management are well-organized and easy to maintain.
  </p>

  <pre class="core-code">
  ├── database
  │   ├── migrations
  │   │   ├── 001_initial.sql
  │   │   └── 002_add_users.sql
  │   ├── seeds
  │   │   └── seed_users.sql
  │   └── db-config.js
  </pre>

  <p class="core-description">
    The `migrations` directory contains SQL files for database schema changes, the `seeds` directory holds scripts for populating the database with initial data, and the `db-config.js` file includes database connection configurations.
